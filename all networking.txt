1. MTU (Maximum Transmission Unit)

MTU is the maximum size of an IP packet that can be transmitted over a network interface without fragmentation.

Default Ethernet MTU

For Ethernet

MTU = 1500 bytes

This means:

Maximum IP Packet Size = 1500 bytes
MTU Example
IP Header = 20 bytes
TCP Header = 20 bytes
Data = 1460 bytes
--------------------
Total = 1500 bytes

So the largest TCP data payload allowed without fragmentation = 1460 bytes.

2. MSS (Maximum Segment Size)

MSS is the maximum amount of TCP data that can be sent in one TCP segment.

It is calculated from MTU.

Formula
MSS = MTU − IP Header − TCP Header

For standard Ethernet:

MSS = 1500 − 20 − 20
MSS = 1460 bytes
MSS Key Points
Item	Value
MTU	1500 bytes
IP Header	20 bytes
TCP Header	20 bytes
MSS	1460 bytes

MSS is negotiated during the TCP handshake.

Example SYN packet:

MSS Option: 1460
3. Ethernet Frame Size

An Ethernet frame contains more than just the IP packet.

Structure:

Preamble        8 bytes
MAC Header     14 bytes
IP Packet    up to 1500 bytes
FCS             4 bytes
Ethernet Frame Components
Field	Size
Preamble + SFD	8 bytes
Destination MAC	6 bytes
Source MAC	6 bytes
EtherType	2 bytes
Payload	1500 bytes
FCS	4 bytes
Why Ethernet Frame Size is 1518 Bytes

The maximum Ethernet frame size (excluding preamble) is:

MAC Header = 14 bytes
Payload = 1500 bytes
FCS = 4 bytes
------------------
Total = 1518 bytes

So:

Max Ethernet Frame = 1518 bytes

DNS RESOLUTION PROCESS
======================

User enters a domain name in browser
Example: www.google.com


STEP 1 — Browser Cache
----------------------

+----------------+
|     Browser    |
+----------------+
        |
        | Check browser DNS cache
        v
If found → Use cached IP
If not found → Go to OS Resolver


STEP 2 — OS DNS Resolver
------------------------

+----------------------+
| Operating System DNS |
|      Resolver        |
+----------------------+
        |
        | Check OS DNS cache
        v
If found → Return IP
If not found → Query Recursive Resolver


STEP 3 — Recursive DNS Resolver
-------------------------------

+------------------------------+
| Recursive DNS Resolver      |
| (ISP DNS / 8.8.8.8 / 1.1.1.1)|
+------------------------------+
               |
               | Query Root Server
               v


STEP 4 — Root DNS Server
------------------------

+-------------------+
|   Root DNS Server |
+-------------------+
        |
        | Response:
        | "Ask .com TLD server"
        v


STEP 5 — TLD Server (.com)
--------------------------

+---------------------+
|   TLD DNS Server    |
|      (.com)         |
+---------------------+
        |
        | Response:
        | "Ask google.com
        | authoritative server"
        v


STEP 6 — Authoritative DNS Server
---------------------------------

+------------------------------+
| Authoritative DNS Server     |
| for google.com               |
+------------------------------+
        |
        | Returns IP address
        v

    www.google.com
         ↓
    142.250.190.14


STEP 7 — Resolver Caching
-------------------------

Recursive resolver caches the response

TTL (Time To Live)
Example: 3600 seconds


STEP 8 — IP Returned to Client
------------------------------

+------------------+
|   Client System  |
+------------------+
        |
        v

Browser receives IP address


STEP 9 — Connection to Server
-----------------------------

Browser connects using:

TCP Port 443 (HTTPS)

+-----------+
      |
      v
+------------------+

| 142.250.190.14   |
+------------------+

Second-Level Domains (SLD)

A Second-Level Domain is the part of the domain name directly below the Top-Level Domain (TLD).

Example:

www.google.com
        |
        |--- .com  → Top Level Domain (TLD)
        |
        |--- google → Second Level Domain (SLD)
        |
        |--- www → Subdomain (3rd level)

                ROOT
                 |
                 .

        +--------+--------+
        |                 |
      TLD               TLD
     (.com)            (.org)

        |                 |
     www.google.com   en.wikipedia.org
